# AGENTS.md

이 파일은 이 저장소에서 작업하는 에이전트를 위한 **짧고 강한 작업 규칙**입니다.

## 1. 가장 먼저 할 일
작업 시작 전에 다음을 먼저 확인합니다.

1. 이슈 설명
2. `WORKFLOW.md`
3. 관련 소스 코드
4. 존재하는 테스트
5. 관련 문서가 있다면 그 문서

## 2. 기본 원칙
- 추측하지 않는다.
- 가장 작은 안전한 수정안을 우선한다.
- 관련 없는 파일은 건드리지 않는다.
- 실패한 검증을 숨기지 않는다.
- 가능한 검증은 반드시 실행한다.
- 실행하지 못한 검증은 이유를 적는다.

## 3. 변경 범위
다음은 특히 보수적으로 다룬다.

- 인증 / 권한
- 결제
- 데이터 삭제
- 보안 관련 로직
- 인프라 / 배포
- 스키마 변경
- 운영 자동화 로직

이 영역은 자동 머지 대상으로 보지 않는다.

## 4. 코드 변경 원칙
- 먼저 원인을 파악한다.
- 원인과 직접 관련된 최소 수정만 시도한다.
- “같이 정리하면 좋아 보이는” 리팩터링은 별도 작업으로 분리한다.
- 한 번에 너무 많은 파일을 건드리지 않는다.

## 5. 검증 원칙
가능하면 아래를 수행한다.

- 관련 테스트
- lint
- 타입체크
- 재현 단계 확인

실패 시:
- 무엇이 실패했는지
- 왜 실패했는지
- 현재 판단상 코드 문제인지 환경 문제인지

를 남긴다.

## 6. 보고 방식
작업 결과에는 최소한 아래가 있어야 한다.

- 무엇을 바꿨는가
- 왜 바꿨는가
- 어떤 검증을 했는가
- 남은 리스크는 무엇인가
- 사람 확인이 필요한가

## 7. 하지 말아야 할 것
- 근거 없이 추정해서 수정하지 않는다.
- 테스트 실패를 무시한 채 완료 처리하지 않는다.
- approval 대기 상태를 임의로 성공 처리하지 않는다.
- 위험 변경을 조용히 확대하지 않는다.

## 8. 이 저장소의 운영 맥락
이 프로젝트는 Codex App Server 기반 orchestration layer다.  
따라서 일반 앱 코드와 달리 아래가 중요하다.

- thread 상태
- retry / backoff
- watcher
- alert suppression
- admin action 안전성
- issue ↔ thread 추적성

즉, 코드가 “돌아간다”만으로 충분하지 않고, **운영 가능성**도 항상 함께 고려해야 한다.
